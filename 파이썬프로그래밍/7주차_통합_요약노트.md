# 📑 파이썬프로그래밍 7주차(7장) 통합 요약 노트

## 1. 튜플 (Tuple)
리스트(List)와 비슷하게 여러 개의 값을 함께 모아 저장할 수 있는 자료형이지만, **한 번 값을 만들면 그 내용을 절대 변경(수정/추가/삭제)할 수 없는** 것이 특징입니다.

### 특징 및 구조
- **소괄호 `()`**와 쉼표(`,`)를 사용하여 만듭니다. (예: `t1 = (1, 2, 3)`)
- 값을 안전하게 보호해야 할 때 주로 사용합니다.
- *주의:* 값이 단 1개뿐인 튜플을 만들 때는 반드시 요소 뒤에 쉼표를 찍어야 합니다. (예: `t2 = (5,)`)
- 값에 접근할 때는 리스트와 동일하게 **대괄호를 이용한 인덱싱**을 사용합니다. (예: `t1[0]`)

### 패킹과 언패킹
- **패킹 (Packing):** 여러 개의 값을 쉼표로 묶어 하나의 튜플 변수에 집어넣는 것.
  - 예: `my_tuple = 1, 2, 3` (소괄호 생략 가능)
- **언패킹 (Unpacking):** 하나의 튜플에 묶여 있는 값들을 풀어서 각각 다른 여러 개의 변수에 한 번에 나누어 저장하는 것.
  - 예: `a, b, c = my_tuple` (a=1, b=2, c=3이 할당됨)

## 2. 딕셔너리 (Dictionary)
단어와 그 뜻이 연결된 사전처럼, **키(Key)와 값(Value)이 한 쌍**으로 이루어진 대응 관계를 표현하는 자료형입니다.

### 특징 및 구조
- **중괄호 `{}`**를 사용하며, 키와 값은 콜론(`:`)으로 묶고 각 쌍은 쉼표(`,`)로 구분합니다.
  - 예: `person = {"이름": "앨리스", "나이": 14}`
- **키(Key):** 고유한 식별자 역할을 하므로 딕셔너리 안에서 중복될 수 없으며, 숫자/문자열/튜플처럼 변경할 수 없는 자료형만 사용할 수 있습니다.
- **값(Value):** 어떤 자료형이든 자유롭게 넣을 수 있으며 변경 가능합니다.

### 딕셔너리 조작하기
- **값 추가/수정:** 대괄호 안에 키를 넣고 `=` 기호로 값을 할당합니다.
  - 예: `person["직업"] = "학생"` (새로운 쌍 추가)
  - 예: `person["나이"] = 15` (기존 나이 값 수정)
- **값 접근하기:**
  1. `딕셔너리[키]`: 가장 일반적인 방법. (키가 없으면 에러 발생)
  2. `딕셔너리.get(키)`: 키가 존재하지 않을 경우 에러를 내지 않고 `None`을 반환하여 안전하게 접근할 때 유용함.
