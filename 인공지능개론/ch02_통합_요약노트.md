## 1. 규칙 기반 모델 (Rule Based Model)
### 개념
- 사람이 판단하는 기준을 `IF-THEN` 형태의 규칙으로 직접 프로그램에 구현한 모델.
- **구성 요소:** 
  - **규칙 엔진:** 문제 해법을 표현하고 실행하는 장치.
  - **규칙 집합:** '만약 X라면 Y를 한다'는 모든 경우의 수의 모음.

### 응용: 챗봇의 유형
1. **규칙 기반 챗봇:** 미리 정의된 키워드 매칭을 통해 답변.
2. **인공지능 기반 챗봇:** 머신러닝/자연어 처리를 통해 스스로 학습하며 답변.
3. **혼합형 챗봇:** 간단한 질의는 규칙으로, 복잡한 대화는 AI 학습 기반으로 처리 (가장 일반적).

### 장점과 한계
- **장점:** 지식 표현이 자연스럽고 구조가 통일되어 있음. 지식과 처리 과정이 분리되어 재사용 가능.
- **한계:** 규칙이 많아지면 논리적 관계가 불분명해짐. 새로운 상황에 대해 스스로 학습하거나 수정할 능력이 없음 (수동 수정 비용 발생).

## 2. 지식 기반 모델과 전문가 시스템
### 지식 기반 모델 (Knowledge Based Model)
- 특정 분야의 전문 지식과 사실, 규칙이 저장된 **지식베이스(Knowledge Base)**를 활용하는 모델.
- 지식(데이터 영역)과 이를 처리하는 인터페이스 엔진이 분리되어 있어 유지보수가 용이함.

### 전문가 시스템 (Expert System)
- **정의:** 특정 분야의 전문 지식을 컴퓨터에 기억시켜 일반인도 전문가 수준의 판단을 내릴 수 있게 돕는 시스템.
- **성립 조건:** 전문가로부터 획득된 지식일 것, 전문적으로 고도의 현실 문제를 다룰 것, 능력이 전문가와 동등할 것.
- **역사적 의의:** AI 성장기에 큰 인기를 끌었으나, 지식 추출의 병목 현상과 유지비 문제로 2차 암흑기를 초래하기도 함.

## 3. 추천 시스템 (Recommender System)
개인화된 맞춤 콘텐츠를 추천하여 사용자 만족도를 높이는 시스템.

### 주요 추천 유형 (필터링 기술)
1. **협업 기반 추천 (Collaborative Filtering):** '유유상종' 원리 이용.
   - **사용자 기반:** 나와 비슷한 성향의 사용자가 구매한 상품 추천.
   - **아이템 기반:** 내가 샀던 아이템과 유사한 아이템(함께 구매되는 빈도가 높은 상품) 추천.
   - **모델 기반:** 머신러닝으로 평점을 예측하여 추천.
2. **콘텐츠 기반 추천 (Content-based Filtering):** 아이템 자체의 특성(음악 장르, 메뉴 재료 등)을 분석하여 추천. 사용자 행동 정보가 적어도 가능.

### 추천 시스템의 문제점
- **콜드 스타트 (Cold Start):** 데이터가 없는 신규 사용자에게는 추천이 어려움 (협업 필터링의 단점).
- **롱테일 (Long Tail) 문제:** 인기 콘텐츠 위주로만 추천되어 비주류 콘텐츠가 소외됨.
- **필터 버블 (Filter Bubble):** 맞춤형 정보만 제공받아 편향된 정보에만 갇히게 되는 현상.

### 활용 사례
- **아마존:** 판매량의 35%가 추천에서 발생. 명시적 평점(별점)과 암묵적 평점(구매 행위) 모두 활용.
- **넷플릭스:** 로그인 순간 사용자의 취향에 맞춰 전체 페이지가 구성되는 수준.
- **페이스북:** 친구 추천 시스템에 적극 활용.
