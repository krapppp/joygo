## 1. LED 제어 심화: 밝기 조절의 원리
단순히 `digitalWrite(HIGH/LOW)`만 사용하는 것이 아니라, 켜져 있는 시간(t_high)과 꺼져 있는 시간의 비율을 조절하여 눈으로 보기에 밝기가 변하는 것처럼 느끼게 할 수 있습니다. (PWM의 기초 원리)

- **핵심 로직:**
  - `HIGH` 상태의 `delay`를 짧게, `LOW` 상태의 `delay`를 길게 하면 어둡게 보임.
  - `HIGH` 상태의 `delay`를 길게, `LOW` 상태의 `delay`를 짧게 하면 밝게 보임.
- **반복문 활용:** `for` 문을 이용하여 `t_high` 값을 0부터 10(또는 100)까지 점진적으로 증가시키면 LED가 서서히 밝아지는 효과를 구현할 수 있습니다.

## 2. C/C++ 기본 자료형
아두이노 프로그래밍의 기초가 되는 C/C++ 자료형의 종류와 특징입니다.

### 정수 및 실수 자료형
- **int (정수):** 4바이트(32비트) 크기. (※ 아두이노 우노 등 8비트 보드에서는 2바이트인 경우도 있으나, 일반적인 C 기준 4바이트).
- **short int / long int:** 더 작거나 큰 정수를 저장.
- **unsigned int:** 0 이상의 양수만 저장하여 표현 범위를 두 배로 넓힘.
- **float (실수):** 4바이트 단정밀도 실수.
- **double (실수):** 8바이트 배정밀도 실수 (아두이노 우노에서는 float과 동일하게 처리되는 경우가 많음).

### 문자 및 문자열 자료형
- **char:** 1바이트 크기. 한 개의 문자를 작은 따옴표(`'A'`)로 감싸서 저장.
- **char* (포인터):** 문자열의 첫 번째 문자가 저장된 **메모리 주소**를 가리킴. 
  - `char name[] = "Alice";` (배열로 저장)
  - `char *ptr = name;` (포인터로 가리킴)
- **sizeof 연산자:** 변수나 자료형이 차지하는 메모리 크기(바이트 단위)를 확인하는 데 사용.

### 기타 자료형
- **const (상수):** 변하지 않는 값을 선언할 때 사용.
- **enum (열거형):** 연관된 상수들의 집합에 이름을 붙여 관리. (예: `Color {RED, GREEN, BLUE}` -> 0, 1, 2 매칭)

## 3. 제어문: for 반복문
정해진 횟수만큼 코드를 반복 실행할 때 사용합니다.

- **기본 형식:** `for (초기식; 조건식; 증감식) { 실행할 코드 }`
- **활용 사례:** 
  - 시리얼 모니터에 0부터 10까지 출력하기.
  - LED 밝기를 0단계부터 10단계까지 순차적으로 변경하기.
  - `while(true)` 문과 조합하여 특정 조건이 만족될 때까지 내부 루프를 반복하고 `break`로 탈출하는 구조로도 활용됨.
