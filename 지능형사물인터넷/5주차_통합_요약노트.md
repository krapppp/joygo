## 1. 배열, 함수, 포인터의 관계
아두이노(C/C++) 프로그래밍에서 메모리(RAM)를 효율적으로 사용하고 복잡한 데이터를 처리하기 위한 핵심 개념입니다.

### 포인터(Pointer)의 개념
- 일반 변수가 데이터 '값'을 저장한다면, 포인터 변수는 데이터가 저장된 **메모리(RAM)의 주소값(위치 정보)**을 저장합니다.
- **주소 연산자(`&`):** 변수의 실제 메모리 주소를 얻을 때 사용합니다. (예: `&val`)
- **참조 연산자(`*`):** 주소가 가리키는 실제 데이터 값에 접근할 때 사용합니다. (예: `*ptr`)

### 포인터가 필요한 이유
1. **메모리 절약 (Call by Reference):** 배열과 같은 큰 데이터를 함수에 넘길 때, 데이터 전체를 복사하지 않고 시작 '주소'만 전달하여 메모리 낭비를 막습니다. (아두이노 UNO의 RAM은 2KB로 매우 적음).
2. **다중 반환값 처리:** C/C++ 함수는 기본적으로 1개의 값만 `return`할 수 있지만, 변수의 주소를 인자로 넘기면 함수 내부에서 여러 변수의 값을 동시에 수정(반환)할 수 있습니다.
3. **배열의 본질:** 배열의 이름(예: `led`)은 사실상 **배열의 첫 번째 요소의 주소를 가리키는 포인터**입니다. 따라서 함수에 `led` 배열을 넘기는 것은 `int *led` 포인터를 넘기는 것과 동일합니다.

## 2. 함수화 및 배열 크기 구하기
반복되는 LED 제어 로직을 함수로 묶고, 배열을 매개변수로 전달하여 코드를 깔끔하게 정리하는 방법입니다.

- **배열 매개변수 전달:**
  `void leds_init(int *led, int led_cnt) { ... }` 
  함수는 배열의 시작 주소(`*led`)와 배열의 크기(`led_cnt`)를 함께 전달받아 처리해야 안전합니다.
- **배열의 크기 자동 계산:**
  `int led_cnt = sizeof(led) / sizeof(led[0]);`
  배열의 전체 메모리 크기를 요소 한 개의 메모리 크기로 나누어 총 개수를 구합니다.

## 3. 코드 모듈화: `.h` 와 `.cpp` 파일 분리
코드가 길어질 때 기능별로 파일을 나누어 관리하고 재사용성을 높이는(라이브러리화) 방법입니다.

### 파일의 역할
- **.h 파일 (헤더 파일):** 프로그램의 구조, 함수, 클래스, 상수 등을 **"선언"**만 하는 곳.
- **.cpp 파일 (소스 파일):** 헤더 파일에 선언된 함수의 실제 동작 로직을 **"구현"**하는 곳.
- **.ino 파일 (메인 파일):** `#include "myLibrary.h"`를 선언하여 분리된 기능들을 불러와 사용합니다.

### C/C++ 전처리기 (Preprocessor)와 헤더 가드
- 컴파일러가 코드를 번역하기 전에 수행하는 지시어(항상 `#`으로 시작).
- **중복 포함 방지 (Header Guard):**
  여러 파일에서 같은 헤더를 `include`할 때 발생하는 재정의 오류(Redefinition Error)를 막기 위해 `#ifndef`, `#define`, `#endif` 패턴을 사용합니다.
  ```cpp
  #ifndef _LEDS_H_  // 만약 _LEDS_H_가 정의되어 있지 않다면
  #define _LEDS_H_  // _LEDS_H_를 정의하고 아래 내용을 포함하라

  void leds_init(int *led, int led_cnt);

  #endif // _LEDS_H_
  ```
