# 📑 정보시스템기반기술 2주차(2장) 통합 요약 노트

## 1. 컴퓨터의 기본 구성과 폰노이만 구조
- **필수 장치:** CPU(중앙처리장치), 메인 메모리
- **주변 장치:** 입력, 출력, 보조 저장장치
- **버스(Bus):** 메인보드 내 장치 간 데이터 통로 (시스템 버스, CPU 버스). 대역폭은 한 번에 전달 가능한 최대 데이터 크기.
- **폰노이만 구조의 한계:** 모든 프로그램은 반드시 메모리에 올라와야 실행 가능하며, CPU와 메모리 간의 속도 차이로 인해 병목현상이 발생할 수 있음.

## 2. CPU와 메모리의 구성
### CPU의 구성
- **산술논리 연산장치(ALU):** 실제 연산 수행.
- **제어장치:** 작업 지시 및 흐름 제어.
- **레지스터(Register):** CPU 내의 임시 저장 공간.
  - *사용자 불가시 레지스터:* 프로그램 카운터(PC), 명령어 레지스터(IR), 메모리 주소 레지스터(MAR) 등.

### 메모리의 종류와 계층 구조
- **휘발성 램(RAM):** DRAM, SRAM.
- **비휘발성 롬(ROM):** 바이오스 등이 저장됨.
- **계층 구조 (속도 빠름 -> 느림):** 레지스터 > 캐시 메모리 > 메인 메모리 > 보조 저장장치. 이를 통해 속도와 용량의 딜레마를 해결함.

## 3. 컴퓨터 성능 향상 기술
CPU와 메모리, 주변장치 간의 속도 차이를 극복하기 위한 다양한 기술입니다.
- **버퍼(Buffer) & 스풀(Spool):** 장치 간 속도 차이를 줄이기 위한 임시 저장 공간. (예: 프린터 스풀러)
- **캐시(Cache):** CPU 내에 위치하여 메인 메모리와의 속도 차이를 완화하는 초고속 메모리. (데이터를 미리 가져오는 프리패치 기능 수행).
- **인터럽트(Interrupt):** 입출력 관리자가 작업을 완료하면 CPU에 보내는 신호. CPU가 입출력 여부를 계속 묻는 '폴링(Polling)' 방식의 비효율성을 극복함.
- **DMA (Direct Memory Access):** CPU의 개입 없이 입출력 장치가 직접 메모리에 접근할 수 있는 권한.
- **멀티프로세싱:** 멀티코어(하나의 프로세서에 여러 코어) 등을 사용하여 여러 명령어를 병렬로 처리함.
