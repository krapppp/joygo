# 📑 정보시스템기반기술 4주차(4장) 통합 요약 노트

## 1. 스레드(Thread)의 개념과 특징
### 스레드란?
- **정의:** 프로세스 내부에서 실처럼 이어지는 실제 **실행 단위**.
- **관계:** 운영체제 입장에서는 프로세스가 작업 단위이지만, 프로세스 내부에서 CPU 작업을 직접 요청하는 것은 스레드임. (작업 > 프로세스 > 스레드)
- **멀티스레딩:** 하나의 프로세스 내부에서 여러 개의 스레드가 동시에 작업을 수행하는 방식.

### 멀티스레딩의 장점
1. **자원 공유:** 프로세스의 코드, 데이터, 파일 자원을 공유하여 메모리 낭비를 줄임.
2. **효율적 문맥 교환:** 프로세스 전환보다 스레드 전환 시 저장/복구할 정보가 적어 속도가 빠름.
3. **응답성 향상:** 특정 스레드가 입출력으로 대기하더라도 다른 스레드는 계속 실행 가능.
4. **적응성:** 멀티코어 CPU 환경에서 스레드를 여러 코어에 분산시켜 최적의 성능을 냄.

---

## 2. 멀티스레딩 모델
스레드를 관리하는 주체(사용자/커널)에 따라 세 가지 모델로 나뉩니다.

1. **사용자 레벨 스레드 (1 to N):**
   - 라이브러리가 스레드를 관리하며 커널은 하나의 프로세스로만 인식.
   - 문맥 교환 비용이 낮지만, 하나의 스레드가 대기(I/O)에 들어가면 전체 스레드가 멈추는 단점이 있음.
2. **커널 레벨 스레드 (1 to 1):**
   - 커널이 직접 스레드를 스케줄링.
   - 대기 상태에서도 다른 스레드가 동작 가능하며 멀티코어 활용에 유리하지만, 문맥 교환 오버헤드가 큼.
3. **멀티레벨 스레드 (M to N):**
   - 사용자 레벨과 커널 레벨의 혼합 모델. 유연하게 작동하지만 구현이 복잡함.

## 3. 프로세스 간 통신 (IPC)
프로세스들이 서로 데이터를 주고받기 위한 기법입니다.

### 통신의 분류
- **방향:** 단방향(Simplex), 양방향(Duplex), 반양방향(Half-duplex).
- **동기화 여부:**
  - **비동기화:** 데이터 도착을 계속 확인하는 '바쁜 대기(Busy Waiting)' 발생.
  - **동기화:** 데이터가 올 때까지 자동으로 대기(Blocking) 상태가 됨 (효율적).

### 주요 통신 기법
- **파이프 (Pipe):** 운영체제가 제공하는 동기화 방식의 단방향 통신. 주로 부모-자식 프로세스 간에 사용.
- **소켓 (Socket):** 네트워크를 통해 다른 컴퓨터에 있는 프로세스와 통신하는 방식. 포트 번호를 사용하여 프로세스를 식별하며 양방향 동기화 통신을 지원함.

### 실습 및 심화 내용
- **`pthread_create()` / `pthread_join()`:** 유닉스 환경에서 스레드를 생성하고 종료를 기다리는 핵심 함수.
- **`kill PID`:** 백그라운드에서 실행 중인 서버 프로세스 등을 강제 종료할 때 사용.
- **파일 포인터:** 파일 내에서 현재 작업 위치(바이트 단위)를 가리키는 지시자.
