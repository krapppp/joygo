# 📑 정보시스템기반기술 3주차(3장) 통합 요약 노트

## 1. 프로세스의 개념과 상태
- **프로그램 vs 프로세스:** 프로그램은 저장장치에 있는 정적인 코드 덩어리이며, 이 프로그램이 실행되어 메모리에 올라오면 **'프로세스(Process)'**가 됨.
- **프로세스의 구조:** 코드 영역, 데이터 영역(정적 데이터), 힙 영역(동적 할당), 스택 영역(지역 변수 등)으로 구성됨.
- **프로세스의 5가지 상태:**
  1. **생성 상태:** 메모리에 올라와 PCB를 할당받은 상태.
  2. **준비 상태:** CPU 할당을 기다리는 상태.
  3. **실행 상태:** CPU를 할당받아 실제 연산을 수행하는 상태.
  4. **대기 상태:** 입출력(I/O) 요청을 하고 완료될 때까지 기다리는 상태.
  5. **완료 상태:** 작업을 마치고 자원을 반납하는 상태.
  *(비활성 상태: 휴식 상태, 보류 상태)*

## 2. 프로세스 제어 블록(PCB)과 문맥 교환
- **PCB (Process Control Block):** 운영체제가 프로세스를 관리하기 위해 필요한 정보(PID, 상태, 프로그램 카운터, 레지스터 정보 등)를 담은 주문서 같은 구조체.
- **문맥 교환 (Context Switching):** CPU가 한 프로세스에서 다른 프로세스로 넘어갈 때, 현재 상태를 PCB에 저장하고 새로운 프로세스의 상태를 읽어오는 과정. (타임 슬라이스 크기에 따라 시스템 성능에 영향을 미침).

## 3. 프로세스 계층구조와 주요 시스템 호출
프로세스는 부모-자식의 트리 구조를 가지며, 이는 자원 회수(가비지 컬렉션)를 쉽게 하기 위함입니다.
- **`fork()`:** 현재 부모 프로세스와 똑같은 자식 프로세스를 메모리에 복제 생성함. (빠른 병렬 처리 목적).
- **`exec()`:** 기존 프로세스(PCB 등)는 유지한 채, 코드와 데이터 영역만 새로운 프로그램으로 완전히 덮어씌움. (메모리 낭비 방지).
- **`wait()`:** 부모 프로세스가 자식 프로세스의 종료(자원 반환)를 기다리는 함수.
  - *고아 프로세스 / 좀비 프로세스:* 자식이 종료되었으나 부모가 자원을 회수하지 않아 발생하는 비정상적 상태.
