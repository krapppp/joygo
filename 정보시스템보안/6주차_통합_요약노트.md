# 📑 정보시스템보안 6주차 통합 요약 노트

## 1. 셸(Shell) 특수문자 및 변수 활용 (5장 중반부)
### 주요 특수문자
셸은 사용자가 입력한 특수문자를 먼저 해석하여 특정 기능으로 변환한 뒤 명령을 실행합니다.
- `*` (별표): 0개 이상의 임의의 모든 문자를 대체 (예: `ls pas*` -> pas로 시작하는 모든 파일).
- `?` (물음표): 길이가 딱 1개인 임의의 문자를 대체.
- `~` (물결표): 사용자의 홈 디렉터리 경로를 의미.
- `;` (세미콜론): 여러 명령어를 한 줄에 순차적으로 실행할 때 사용.

### 셸 변수와 환경 변수
- **셸 변수 (지역 변수):** 현재 실행 중인 셸에서만 유효하며 서브 셸로 전달되지 않음.
- **환경 변수 (전역 변수):** 현재 셸뿐만 아니라 서브 셸에도 전달됨. 일반적으로 **대문자**로 표기함 (예: `PATH`, `HOME`, `USER`).
- **관련 명령어:**
  - `set`: 셸 변수와 환경 변수 모두 출력.
  - `env`: 환경 변수만 출력.
  - `export [변수명]`: 기존 셸 변수를 환경 변수로 승격시키거나, 새로 환경 변수를 설정함.
  - 변수를 출력할 때는 앞에 `$`를 붙임 (예: `echo $USER`).

## 2. 입출력 방향 변경 (Redirection)
리눅스의 표준 입출력(키보드 입력, 모니터 출력) 방향을 파일로 돌리는 기능입니다.
- **표준 출력 덮어쓰기 (`>`):** 명령의 정상 실행 결과를 화면 대신 파일에 덮어써서 저장. (예: `ls > list.txt`)
- **표준 출력 추가하기 (`>>`):** 기존 파일 내용을 지우지 않고 맨 아래에 결과를 이어서 추가.
- **표준 오류 변경 (`2>`):** 에러 메시지만 따로 파일에 저장. (정상 출력은 `1>`, 에러는 `2>` 디스크립터를 사용함).
- **표준 입력 변경 (`<`):** 키보드 대신 특정 파일의 내용을 입력값으로 받음.

## 3. 앨리어스, 히스토리 및 프롬프트 설정
### 앨리어스 (Alias)와 히스토리 (History)
- **`alias`:** 길거나 복잡한 명령어를 짧은 별칭으로 지정함. (예: `alias ll='ls -alF'`). `=` 양옆에 공백이 없어야 함.
- **`unalias`:** 설정된 앨리어스 삭제.
- **`history`:** 이전에 입력했던 명령어 목록 출력.
  - `!!` : 바로 직전 명령 재실행.
  - 키보드 위/아래 방향키(`↑`, `↓`)를 눌러 이전 명령을 불러와 편집 후 실행 가능.

### 프롬프트와 환경 설정 파일
- **프롬프트 설정 (`PS1`):** 명령 입력 대기 상태를 나타내는 프롬프트 모양은 `PS1` 환경 변수를 수정하여 바꿀 수 있음. (이스케이프 문자 `\u`(사용자), `\w`(현재 작업 디렉터리) 등 활용).
- **환경 설정 파일:**
  - **시스템 환경 파일:** `/etc/profile`, `/etc/bash.bashrc` 등 관리자만 수정 가능하며 모든 사용자에게 적용됨.
  - **사용자 환경 파일:** `~/.profile`, `~/.bashrc`, `~/.bash_aliases` 등 사용자 홈 디렉터리에 숨김 파일로 존재. 로그인 시 실행되어 개별 환경을 구축함. (수정 후 즉시 적용하려면 `source ~/.profile` 실행).
