## 1. 리눅스 문서 편집기 심화 (4장 후반)
### vi 편집기 환경 설정
- **기본 환경 설정 명령 (마지막 행 모드):**
  - `:set nu` : 각 행의 앞에 행 번호를 표시함.
  - `:set nonu` : 행 번호 표시를 숨김.
  - `:set list` : 눈에 보이지 않는 특수문자(행 끝 `$`, 탭 `^I` 등)를 화면에 표시함.
  - `:set` : 현재 설정되어 변경된 vi 환경 설정 값을 모두 출력함.
- **환경 설정 영구 적용 방법:**
  1. **.exrc 파일 사용:** 사용자의 홈 디렉터리(`~`)에 `.exrc` 파일을 생성하고 설정값을 적어두면, vi를 실행할 때마다 자동으로 해당 설정이 적용됨 (가장 보편적인 방법).
  2. **환경 변수 사용:** `EXINIT` 환경 변수에 설정값을 지정하여 적용 가능.

### nano 편집기 사용법
- **특징:** 모드형인 vi와 달리, 윈도우의 메모장처럼 직관적으로 입력할 수 있는 **비모드형 편집기**.
- GNU 프로젝트에서 `pico` 편집기를 기반으로 개발한 오픈소스 편집기이며, 우분투에 기본 내장되어 초보자가 사용하기 편리함.
- 화면 하단에 `^O` (저장), `^X` (종료) 와 같이 단축키(Ctrl 키 조합) 메뉴가 항상 표시되어 있어 쉽게 조작할 수 있음.

## 2. 셸(Shell)의 기능과 종류 (5장 1절)
셸은 사용자가 입력한 명령을 해석하여 리눅스 커널(Kernel)에 전달하는 중계자(명령어 해석기) 역할을 합니다.

### 셸의 3대 주요 기능
1. **명령어 해석기 기능:** 
   - 프롬프트(`$`, `#`)에서 사용자의 입력을 받아 내장 명령이면 즉시 실행하고, 외부 명령이면 자식 프로세스를 생성하여 실행함.
2. **프로그래밍 기능:** 
   - 여러 명령을 조합하여 반복적인 작업을 자동화할 수 있는 **셸 스크립트(Shell Script)** 작성 기능 지원.
3. **사용자 환경 설정 기능:** 
   - 로그인 시 초기화 파일을 읽어들여 명령어 경로(PATH), 환경 변수, 기본 권한 설정 등 사용자 맞춤 환경을 구성함.

### 셸의 주요 종류
- **본셸 (Bourne Shell, `sh`):** 최초의 유닉스 셸. 빠르지만 히스토리나 앨리어스 등 편의 기능이 부족함.
- **C셸 (`csh`):** 빌 조이가 개발. C 언어와 유사한 문법을 가지며 본셸보다 다양한 편의 기능을 추가함.
- **콘셸 (`ksh`):** 데이비드 콘이 개발. 본셸과 호환성을 유지하면서 C셸의 장점을 수용함.
- **배시셸 (Bash Shell, `bash`):** GNU 프로젝트로 개발된 리눅스의 **기본 셸** (Bourne Again Shell). 본셸, C셸, 콘셸의 장점을 모두 통합함.
- **대시셸 (Dash Shell, `dash`):** 크기가 작고 처리 속도가 빠른 셸. 우분투 등에서 시스템 셸(부팅 스크립트용)로 주로 사용됨.

### 셸 기본 사용법
- **로그인 셸:** 사용자가 로그인할 때 자동으로 실행되는 기본 셸. `/etc/passwd` 파일에서 각 사용자별 지정 셸을 확인할 수 있음.
- **서브 셸:** 로그인 셸(프롬프트) 안에서 다른 셸을 실행하여 생성된 자식 셸. `exit`나 `Ctrl + D`로 종료하면 이전 셸로 돌아감.
- **셸 변경 (`chsh`):** 
  - 사용자의 기본 로그인 셸을 변경하는 명령어. (예: `chsh -s /bin/dash user1`)
  - 변경 가능한 셸의 목록은 `/etc/shells` 파일에 기록되어 있음. 절대 경로를 정확히 입력해야 정상 변경됨.
- **셸 내장 명령:** 
  - `cd`와 같이 셸 내부에 포함되어 있어 별도의 실행 파일(바이너리)이 존재하지 않는 명령. (`file` 명령어로 실행 파일 유무 확인 가능).
