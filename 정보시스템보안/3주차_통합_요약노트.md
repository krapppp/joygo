# 📑 정보시스템보안 3주차 통합 요약 노트

## 1. 리눅스 파일 시스템 구조
### 파일 시스템의 구성 요소
리눅스는 시스템과 사용자의 파일을 체계적으로 관리하기 위해 다음 요소들을 사용합니다.
- **슈퍼 블록 (Super Block):** 파일 시스템의 전체 상태 정보, 블록 크기, inode 및 데이터 블록의 총 개수 등을 저장.
- **inode (Index Node):** 파일의 속성 정보(파일 종류, 크기, 소유자, 접근 권한, 수정 시간 등)와 실제 데이터 블록의 메모리 주소를 저장하는 핵심 구조. (`ls -l` 명령어로 확인되는 정보들)
- **데이터 블록 (Data Block):** 파일의 실제 데이터가 저장되는 물리적 공간. (직접/간접/이중 간접 블록 방식으로 주소 할당)

### 리눅스 주요 파일 시스템 종류
- **ext 계열:** ext2, ext3(저널링 도입), ext4(1EB 이상 볼륨 지원, 현재 범용적으로 사용됨).
- **XFS:** 대규모 데이터 관리에 적합한 고성능 저널링 64비트 파일 시스템.
- **가상 파일 시스템:** 물리적 디스크가 아닌 메모리에서 특정 용도로 임시 생성/소멸됨.

## 2. 파일의 종류와 디렉터리 계층 구조
### 파일의 종류
- **일반 파일:** 텍스트 문서, 실행 파일, 이미지 파일 등.
- **디렉터리 파일:** 하위 디렉터리와 파일들의 정보를 담고 있는 폴더 (리눅스에서는 디렉터리도 파일로 취급).
- **심볼릭 링크:** 윈도우의 '바로가기'와 같은 역할.
- **장치 파일:** 키보드, 하드디스크 등 하드웨어 장치를 관리하는 파일 (주로 `/dev`에 위치).

### 디렉터리 구조와 경로
- **루트(Root) 디렉터리:** 최상위 디렉터리로 `/` 로 표기. (아래에 `/etc`, `/usr`, `/home` 등이 위치함).
- **홈(Home) 디렉터리:** 사용자별 할당 공간. `~` 기호로 표기 (예: `~user1`).
- **절대 경로:** 항상 루트(`/`)부터 시작하여 목표 파일까지의 전체 경로를 명시하는 방법.
- **상대 경로:** 현재 작업 중인 디렉터리(`.`)를 기준으로 목표 위치를 명시하는 방법. 상위 디렉터리는 `..`으로 표기.

## 3. 디렉터리 및 파일 관리 기초 명령어
### 디렉터리 관련 명령
- **`pwd`:** 현재 작업 중인 디렉터리의 절대 경로를 출력.
- **`cd [경로]`:** 지정한 디렉터리로 이동. (`cd ~` 또는 `cd`만 입력하면 홈 디렉터리로 이동).
- **`ls [옵션] [경로]`:** 디렉터리 내의 파일 목록 확인.
  - `-a`: 숨김 파일(점`.`으로 시작)까지 모두 출력.
  - `-l`: 파일의 상세 정보(권한, 소유자, 용량, 시간 등) 출력.
  - `-F`: 파일 종류 표시 (`/`는 디렉터리, `*`는 실행파일 등).
- **`mkdir [옵션] [이름]`:** 새 디렉터리 생성. (`-p` 옵션 시 하위 경로까지 한 번에 생성).
- **`rmdir [이름]`:** 빈 디렉터리 삭제.

### 파일 내용 확인 명령
- **`cat`:** 파일 전체 내용을 한 번에 화면에 출력. (`-n` 옵션으로 행 번호 표시).
- **`more` & `less`:** 화면 단위로 끊어서 출력. (`less`가 스크롤 이동 및 검색 기능 지원 등 더 강력함).
- **`head` / `tail`:** 파일의 앞부분 / 뒷부분(기본 10줄)만 출력.
  - `tail -f`: 파일 내용이 업데이트될 때마다 실시간으로 출력 (로그 확인에 유용).

### 파일 복사 명령
- **`cp [원본] [목적지]`:** 파일이나 디렉터리를 복사.
  - 목적지가 파일명이면 그 이름으로 복사, 디렉터리면 해당 디렉터리 안으로 복사됨.
  - `-i`: 덮어쓸 때 미리 확인 메시지 출력.
  - `-r`: 디렉터리와 그 안의 하위 내용까지 모두 통째로 복사.
