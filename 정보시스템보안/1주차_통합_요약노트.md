# 📑 정보시스템보안 1주차 통합 요약 노트

## 1. 운영체제 개요
### 운영체제(OS)의 역할과 기능
- **역할:** 컴퓨터 하드웨어와 응용 프로그램 간의 인터페이스 제공, 사용자에게 편리한 환경 제공, 시스템 자원(CPU, 메모리 등)의 효율적 관리.
- **주요 기능:** 프로세스 관리(생성/스케줄링), 메모리 관리, 파일 시스템 관리, 입출력 관리, 자원 보호 및 사용자 인터페이스(GUI/CLI) 제공.

### 운영체제의 구성 요소와 동작 모드
- **구성 요소:** 커널(핵심 제어), 파일 시스템, 디바이스 드라이버, 시스템 호출(System Call), 사용자 인터페이스.
- **CPU 동작 모드:**
  - **커널 모드 (슈퍼바이저 모드):** 하드웨어 자원에 직접 접근 가능, OS 핵심 기능 수행.
  - **사용자 모드 (보호 모드):** 하드웨어 직접 접근 불가, 응용 프로그램이 실행되는 모드로 시스템 호출을 통해 자원 요청.

## 2. 유닉스와 리눅스의 역사 및 특징
### 유닉스(UNIX)의 발전
- 1969년 AT&T 벨 연구소에서 켄 톰슨과 데니스 리치가 개발.
- 1973년 C 언어로 재작성되어 최초의 고급 언어 운영체제가 됨.
- **특징:** 대화형 시스템, 다중 사용자/다중 작업 지원, 높은 이식성과 확장성, 계층적 트리 파일 시스템 구조.

### 리눅스(Linux)의 탄생과 특징
- 1991년 리누스 토르발스가 유닉스와 완벽하게 호환되는 공개 소프트웨어 커널로 발표.
- **주요 특징:** 
  - **오픈소스:** 누구나 무료로 다운로드, 수정, 재배포 가능.
  - **완벽한 유닉스 호환:** 유닉스 명령어 및 시스템 콜 표준 준수.
  - 서버용 및 데스크톱용으로 뛰어난 안정성과 강력한 네트워크/보안 기능 제공.
- **구조:** 커널(핵심 제어), 셸(명령어 해석기, 기본은 Bash 셸), 응용 프로그램으로 구성.

### GNU 프로젝트와 GPL 라이선스
- **GNU:** "GNU is Not Unix"의 약자로, 소프트웨어 독점에 반대하고 자유로운 사용을 보장하는 프로젝트 (리처드 스톨먼 주도).
- **GPL (General Public License):** 프로그램 사용, 수정, 재배포의 자유를 보장하며, 수정된 코드 배포 시 원본 소스 코드도 반드시 공개해야 함을 명시.

## 3. 리눅스 배포판 (Ubuntu)
- 리눅스 커널과 GNU 응용 프로그램을 묶어 사용자가 쉽게 설치할 수 있도록 만든 패키지. (예: 데비안, 레드햇, 슬랙웨어 계열)
- **우분투 (Ubuntu):**
  - 데비안(Debian) 계열의 리눅스 배포판.
  - "전 세계 누구나 어렵지 않게 리눅스를 사용하자"는 철학으로 캐노니컬사에서 개발.
  - 본 강의 실습 환경: **우분투 24.04 LTS** (2024년 4월 출시, Long Term Support 버전으로 5년 지원).

---

## 4. 리눅스 실습 환경 구축
본격적인 실습을 위해 기존 PC에 가상머신을 설치하고 그 위에 우분투를 설치합니다.

### VMware Workstation Pro 설치
- **가상머신:** 호스트 OS(윈도우) 내에 가상의 PC 환경을 만들어 게스트 OS(리눅스)를 실행할 수 있게 해주는 프로그램.
- 브로드컴 사이트에서 무료 개인용 버전(VMware Workstation Pro 17.5.x)을 다운로드하여 설치.

### 우분투 24.04 리눅스 설치 과정
1. **ISO 다운로드:** 공식 홈페이지에서 `ubuntu-24.04-desktop-amd64.iso` 다운로드.
2. **VMware 설정:** 새 가상머신 생성 시 Linux -> Ubuntu 64-bit 선택 (디스크 20GB 할당).
3. **OS 설치:** 가상머신 CD/DVD 설정에 ISO 파일을 넣고 부팅.
4. **기본 설정:** 한국어 키보드 세팅, 디스크 지우고 Ubuntu 설치 선택, 사용자 계정 생성 및 재부팅 후 초기 설정 완료.
