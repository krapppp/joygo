# 📑 정보시스템보안 7주차 통합 요약 노트

## 1. 파일 속성과 접근 권한 기초 (6장 전반부)
리눅스 시스템은 다중 사용자 환경이므로, 파일이나 디렉터리를 생성할 때 소유자, 그룹, 기타 사용자를 분리하여 각각 접근 권한을 철저히 관리합니다.

### 접근 권한의 종류
- **읽기 (`r`, Read):** 파일의 내용을 보거나(cat), 디렉터리의 파일 목록을 볼 수 있는 권한(ls).
- **쓰기 (`w`, Write):** 파일의 내용을 수정/삭제하거나, 디렉터리 내에 파일을 생성/삭제할 수 있는 권한.
- **실행 (`x`, eXecute):** 실행 파일이나 셸 스크립트를 실행할 수 있는 권한. 디렉터리의 경우 해당 디렉터리로 이동(cd)하기 위해 반드시 필요한 권한.

### 권한 표기 방법 (`ls -l` 기준)
- `drwxr-xr-x`와 같은 형태로 10자리 문자가 출력됨.
- 맨 앞자리: 파일 종류 (`-`는 일반 파일, `d`는 디렉터리, `l`은 심볼릭 링크).
- 나머지 9자리: 3개씩 끊어서 **소유자(User) / 그룹(Group) / 기타 사용자(Other)**의 `rwx` 권한 유무를 나타냄. 권한이 없으면 `-`로 표시.

## 2. 파일 접근 권한 변경 (`chmod`)
오직 파일의 소유자나 시스템 관리자(root)만 사용할 수 있습니다.

### 문자 모드 (Symbolic Mode)
- **사용자 카테고리:** `u` (소유자), `g` (그룹), `o` (기타), `a` (모두)
- **연산자:** `+` (권한 추가), `-` (권한 제거), `=` (해당 권한으로 덮어쓰기 설정)
- **사용 예시:** 
  - `chmod u+x file.txt` (소유자에게 실행 권한 추가)
  - `chmod go-w file.txt` (그룹과 기타 사용자에게 쓰기 권한 박탈)

### 숫자 모드 (Numeric Mode)
각 권한을 이진수에서 10진수 숫자로 환산하여 더 빠르고 직관적으로 권한을 세팅합니다.
- **읽기(`r`): 4, 쓰기(`w`): 2, 실행(`x`): 1**
- 권한을 더해서 부여함. (예: `rwx` = 4+2+1 = 7, `rw-` = 4+2 = 6, `r-x` = 4+1 = 5)
- **사용 예시:** 
  - `chmod 755 file` (소유자:7(rwx), 그룹:5(r-x), 기타:5(r-x))
  - `chmod 644 file` (소유자:6(rw-), 그룹:4(r--), 기타:4(r--))

## 3. 기본 접근 권한과 특수 접근 권한
### 기본 접근 권한 설정 (`umask`)
파일이나 디렉터리를 새로 생성할 때 기본적으로 제외할(주지 않을) 권한 마스크 값을 설정합니다.
- 파일 생성 기본 최대 권한은 666(rw-rw-rw-), 디렉터리는 777(rwxrwxrwx)입니다.
- 기본 umask가 `002`라면: 파일은 666 - 002 = 664(rw-r--r--) 권한으로 생성되고, 디렉터리는 777 - 002 = 775(rwxrwxr-x) 권한으로 생성됨.

### 특수 접근 권한
시스템의 유연한 관리와 보안을 위해 특정 조건에서 특별하게 동작하는 권한입니다. 숫자 모드 설정 시 기존 3자리 앞에 4번째 자리(1,2,4)로 덧붙여 설정합니다.
1. **SetUID (4000):** 실행하는 동안 파일의 소유자 권한을 임시로 획득함. (예: `/usr/bin/passwd` 실행 시 임시로 root 권한을 얻어 비밀번호 변경). 소유자 권한 자리에 `s`로 표시.
2. **SetGID (2000):** 실행하는 동안 파일의 그룹 권한을 획득함. 그룹 권한 자리에 `s`로 표시.
3. **스티키 비트 (Sticky Bit, 1000):** 주로 디렉터리에 적용되며(예: `/tmp`), 누구나 파일을 만들 수 있지만 삭제는 파일의 소유자(본인)나 관리자만 가능하도록 보호함. 기타 사용자 권한 자리에 `t`로 표시.
  - *주의:* 실행 권한(`x`)이 없는 상태에서 특수 권한만 설정하면 대문자 `S`나 `T`로 표시되며 정상 동작하지 않음.
