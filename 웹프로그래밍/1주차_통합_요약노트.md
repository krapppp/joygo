# 📑 웹프로그래밍 1주차 통합 요약 노트

## 1. 웹(World Wide Web)의 기본 이해
### 웹의 목적 및 구성
- **기본 목적:** 여러 컴퓨터에서 문서를 공유하고 열람하기 위함.
- **웹의 구조:** 인터넷을 통해 거미줄처럼 연결된 정보망(WWW).
- **핵심 구성:** 
    - **웹 서버 (Web Server):** 웹 사이트를 탑재하고 클라이언트의 요청에 따라 데이터를 전송 및 관리하는 컴퓨터(소프트웨어 포함).
    - **웹 클라이언트 (Web Client):** 사용자 인터페이스를 담당하며, 서버에 문서를 요청하고 결과를 출력하는 환경(예: 웹 브라우저).

### 인터넷 vs 웹
- **인터넷:** 1969년 ARPA에서 시작된 전 세계적인 컴퓨터 연결 네트워크 (고속도로에 비유).
- **웹 (WWW):** 인터넷을 활용하는 응용 서비스 중 하나 (고속도로를 이용한 물류 산업에 비유).

## 2. 웹 브라우저 (Web Browser)
### 최초의 브라우저 및 역사
- **WorldWideWeb:** 1990년 팀 버너스리(Tim Berners-Lee)가 개발, 이후 **Nexus**로 개명.
- **주요 역사:** Mosaic(1993) -> Netscape Navigator(1994) -> IE(1995) -> Chrome(2008) -> Edge(2015).

### 주요 브라우저 특징
- **Netscape Navigator:** 최초의 GUI 기반 대중적 브라우저.
- **Internet Explorer:** MS 윈도우 번들을 통해 시장을 잠식했으나 현재는 중단됨.
- **Mozilla Firefox:** W3C 표준안에 가장 충실하며 Netscape의 소스 공개로 탄생.
- **Google Chrome:** 현재 가장 높은 시장 점유율을 차지하는 브라우저.

## 3. 웹 사이트 구축 및 소프트웨어
### 웹 사이트 구성 요소
1. **웹 서버 컴퓨터:** 하드웨어 장치.
2. **웹 서버 소프트웨어(HTTPd):** 클라이언트 요청 해석 및 결과 전송.
3. **웹 페이지/데이터베이스:** 저장 및 관리되는 실제 데이터.
4. **웹 서버 응용프로그램:** 로그인, 검색 등 복잡한 비즈니스 로직 처리.

### 서버 소프트웨어 및 개발 언어
- **서버 소프트웨어 종류:** Apache, IIS, NGINX, GWS(Google).
- **개발 언어:** Node.js(자바스크립트), JSP(Java), PHP, Python, C/C++ 등.

## 4. 웹 문서의 특성
- **전자 문서와 차이점:** 전자 문서는 모든 요소를 한 파일에 저장하지만, **웹 문서(HTML)**는 페이지 단위로 분할 저장되며 하이퍼링크를 통해 상호 연결됨.
- **하이퍼링크(Hyperlink):** 다른 웹 페이지의 주소를 가진 텍스트/이미지 정보로, 사용자가 읽는 순서를 직접 결정(내비게이션)하게 함.

## 5. 웹 페이지의 주소 (URL) 및 통신 (HTTP)
### URL (Uniform Resource Locator) 구조
- **형식:** `프로토콜://서버주소:포트번호/경로명/파일명`
- **예시:** `http://www.oracle.com:80/tech/index.html`
    - **프로토콜:** HTTP, HTTPS, FTP 등.
    - **서버 주소:** IP 주소 또는 도메인 네임.
    - **포트 번호:** 서버가 대기 중인 채널(HTTP 기본 80, HTTPS 443).

### HTTP 통신 과정 (HTTP 세션)
1. **연결 요청:** 클라이언트가 서버에 연결 시도.
2. **연결 수락:** 서버가 요청을 승인.
3. **문서 요청:** 클라이언트가 특정 HTML 페이지 요청.
4. **문서 읽기:** 서버가 저장소에서 해당 파일 검색.
5. **문서 전송:** 서버가 클라이언트에게 데이터 전송.
6. **해독 및 출력:** 브라우저가 HTML을 해석하여 사용자에게 화면 출력.
